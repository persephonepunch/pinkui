Being AGILE

Being AGILE, has it's roots in programming. I was discussing process to somebody, and they asked me what role I played in AGILE. This threw me off, and then I realized they wanted to describe the role as a SCRUM process that Program Management takes.

AGILE, Version Control and Code Commits are an intrinsic part of Github. Deployment that is automated Vercel/Netlify is "hooked" to this non destructive work flow.

Semantic ARchitecture

Login and Invitation

Netlify CMS is a headless CMS that can host any Static / Collections, Pages, Templates, and UI/UX (CSS, JS). It has config and functions management tools that are linked to real time APIs, and GITHUB based file storage.

Invitations to edit/collaborate are free, and token based connectivity to GITHUB public or private, eliminates the database requirement.

Deployment to free to use/scale Vercel/Netlify with free SSL makes this "best practice" particularly good for E Commerce with PCI security restrictions.

Additional tools: Source Set Image uploading, Time Stamping, Role Based Access Control, Automation, Edge (Publishing Globally to near usability), and Unlimited SSL secure File Naming, makes publishing simple, secure, scaleable. Usage for Personalization, Globaliztion, CDN hosting for images (trackable pixel for BI/Analytics), is ideal for Strategy setups and emerging ideation.

Ability to publish and compact code/css/js in a CDN that can serve UI/UX to multiple channels and cross CMS/CRM/ERP also makes this free to use instant set up appealing. Usage for Designers, Marketers and especially Programmers is very immediately clear.
Logins and Password Retrieval is invitation based and can be linked to Github Secure Access/Token Entitlement. It is fully moderated and is compliant to PCI/HIPAA regulation.
