Core IA

Things you need in addition to "just the ui". Foundation Usability that should be included in User Testing and Code Review.

  • CSS Reset

    Normalize CSS with core rules that make different browsers renders consistent. Common Libraries: Bootstrap, Design Systems will have this built into the "Global" Standard. A Builder like Webflow will include the core reset classes in the Vanilla JS/CSS/HTML export.
    Normalize CSS

  • Position Properties

    Nest Divs to accommodate Images, and Framing that Progressively Scales with Elegant Design that doesn't depend on Cropping Images for Media Breaks. Div / Section Based Framing includes: static, relative, fixed, absolute, sticky. An image inside an absolute box in a relative div, will give you a 100% 100% image that progressively scales with media breaks. The advantage of this is: you can include ADA and Semantic Searchable tagging with images for search and ADA compliance setup. If you are on "Legacy" E Commerce like Site Genesis for Salesforce Commerce, you will have to utilize a particular amount of effort, organization, when loading multiple images for Media Breaks, at penalty to page load and screen readers. Building a CDN that supports Progressive Image Loading is critical to making shift to SFRA.
    Position CSS

  • W3C Compliance

    ADA compliance means that your page (regardless of way it compiles-React, VUE) must include tagging to make it screen legible. If your pages compile without headers that include meta tagging instructions for machine read and your images don't include tagging, you will fail compliance checklists. This means your ASSETs IMAGES need ALT tags in Image Loading Publishing and Page Scan.

    The penalty for non compliance for e commerce is actively regulated.
    w3c Compliance
